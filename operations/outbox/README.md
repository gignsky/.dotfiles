# Operations Outbox

## Purpose
This directory contains items that have been reviewed and approved by Captain, ready for agent processing.

## Workflow
Items moved here from `../inbox/` indicate:
- Captain has reviewed the content
- Any necessary annotations/notes have been added
- The item is approved for implementation/action
- Assigned agents should process when directed

## Processing Protocol
- Agents check this directory when instructed to process approved items
- Items should be moved to appropriate final locations after processing
- Create completion logs documenting actions taken

## File Organization
Approved items maintain their original naming but may include:
- Captain's annotations within the file
- Processing assignments in file metadata
- Implementation priority indicators
